# Relationship Overview

## Core structure
- A **Period** is a large chapter of life (for example: Childhood, Military Service, Family Years).
- A **Period** contains many **Events**.
- An **Event** is a specific moment or episode.

## Content inside an event
- **Memories** are narrative records (transcript/text context).
- **Narrations** are recorded audio stories tied to an event.
- **Assets** are supporting files (photos, documents, audio files).

## Practical model
- Think of the app as: **Periods -> Events -> Evidence**.
- Evidence for an event is mainly:
  - Memory/narrative context
  - Narration audio
  - Assets (photos/documents/files)

## Unlinked items
- If an asset is uploaded without an event, it goes to the **Unlinked Assets Inbox** until you link it.

## In one sentence
- **Periods organize Events, and Events hold the memories/narrations/assets that describe what happened.**
